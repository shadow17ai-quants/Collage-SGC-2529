9. What I Learned From the Integrated Portfolio Program
------------------------------------------------------------------------
* Architecture Alignment: Implementing a terminal system requires a modular 
  mindset. Separating UI logic (show_menu) from structural processing 
  (view_portfolio) keeps the code maintainable.
* Data Association: Since C lacks native objects, multiple characteristics of 
  an entity (e.g., Stock Symbol, Price, Quantity) must be tracked using 
  parallel arrays linked by a shared index (`i`).
* Input Sanitization is Non-Negotiable: Interfacing raw user input with underlying 
  logic requires continuous clearing of the input buffer (`while(getchar() != '\n')`) 
  and formatting normalization (`toupper`) to prevent crashes or silent failures.


10. Core Questions About Pointers (Phase 4 Preparation)
------------------------------------------------------------------------
1. Tracking the Map: If a pointer simply holds a memory address, how does the 
   compiler know how many bytes to read or modify when dereferencing an `int*` 
   versus a `char*`?
2. Pass-by-Reference Context: We've used pointers implicitly in `scanf("%d", &val)`. 
   How does passing an address to a custom function structurally alter execution 
   memory compared to traditional pass-by-value?
3. Array-Pointer Equivalence: It's said that array names act like pointers to 
   their first element. What is the explicit architectural difference between a 
   declared array variable and a dedicated pointer variable?
